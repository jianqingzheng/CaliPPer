## Distance Metric Variants

Each figure has two panel sets in separate subfolders:

| Subfolder | Distance | Transform | Status |
|-----------|----------|-----------|--------|
| `lev-logtransf/` | Levenshtein ratio | `log(k*(1-ratio+b))` | COMPLETE — all 311 panels preserved |
| `blosum-sqrt/` | BLOSUM-SW (BLOSUM62) | `sqrt(1-sim)` | IN PROGRESS — TCR first, BCR later |

**Config:** `Manuscript/designed_figures/panels/dist_config.py` — set `DIST_TYPE` env var to switch.

Panels in figure root directories are the CURRENT working set. `lev-logtransf/` is the archived baseline.

## Method & Settings

| Setting | Value |
|---------|-------|
| **Method** | S2DD v2.7, `General_Eval/s2dd.py` (canonical) |
| **Prediction** | VBIAS_BETA_LAM=0.05, adaptive bins, per-set binning |
| **Recalibration** | CALIBRATION_LAM=0.0, adaptive theta/n_bins, sigma_C distances |
| **TCR epitope distance** | uniform+znorm_sum (`compute_s2dd_distances(task='subset')`) |
| **TCR distance / all BCR** | sigma_C+weighted_max_znorm (degradation strategy) |
| **Baselines** | PAPE + M-CBPE, 3-chain naive Levenshtein (aligned 2026-04-28) |
| **TCR CT cal** | v3+v4 only (seen/unseen/McPAS/IEDB excluded from calibration) |
| **BCR CT** | fold4-as-cal (fold95 model, fold0/1/2/3 training). Data: `results/bcr_bind_ct_fold4cal/` |
| **BCR model** | 600 AA XBCR-net (retrained 2026-04-27 for flu HA inclusion) |

## Visual Style

- Square aspect for scatter panels; explicit r/p/MAE/n stats
- Colorbar capped at 0.15 with `extend='max'` for heatmaps; grey for missing
- Strip overlay on boxplots + Wilcoxon significance brackets
- Colors: S2DD=orange, PAPE=blue, M-CBPE=grey (consistent across all panels)
- DPI=300, font sizes from `style_config.py`

## Organization

```
panels/
├── PANEL_MANIFEST.md          ← this file (single source of truth)
├── fig2/
│   ├── <panel>.pdf + .png     ← content-named panel files
│   └── scripts/
│       ├── README.md          ← maps scripts → panels generated
│       └── *.py               ← generator scripts
├── fig3/ (same structure)
├── fig4/ (same structure)
├── fig5/ (same structure)
└── fig6/ (same structure)
```

**Rules:**
- Each panel filename describes content only (no `a/b/c/...` indices) so assembly can rearrange freely
- Each `fig{N}/scripts/README.md` maps every script to the panels it produces
- All panels are computed from source data — zero hardcoded values
- Prerequisite caches live in `results/fig2_cache/`, `results/fig2_bcr_cache/`, `results/fig3_fig4_bcr_cache/`
